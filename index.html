MkDocs Versioned#

The mkdocs-versioned project adds the ability to build multiple versions of MkDocs documentation projects. This is a popular requirement, if, for example you want to set both development docs and those for the stable release. Or, you may still support an old release and have users using it.

Warning

This work is experimental and should be viewed as a very early alpha, that sortof works.

Usage#

Simply install with pip, pip install mkdocs-versioned and then use the command mkdocs_versioned. This command is similar to mkdocs build, view the --help output to see some of the configuration options.

By default, it will attempt to build all versions, and it will build the latest docs to both / and /latest. To show a specific version at the root, use mkdocs_versioned --default=0.13.0. Replacing 0.13.0 with the version you want to use.

Warning

For mkdocs-versioned to work, it assumes that each of the branches can be built with the installed version of Mkdocs.
